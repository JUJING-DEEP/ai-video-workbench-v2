# AI Video Workbench Product Specification

## Purpose

AI Video Workbench is a local production workspace for turning AI-generated
storyboard scripts into structured short-video production plans. It focuses on
parsing storyboard text, managing shot data, tracking asset readiness, generating
subtitles, validating timelines, and preparing render plans.

## Core Users

- Short-video creators using AI tools for scripts, images, voiceovers, and video clips.
- Operators who need a repeatable way to manage hundreds of storyboard shots.
- Editors who need clear asset status, timing, subtitle, and render-readiness checks.

## v1.0.0-beta.2 Scope

- Import storyboard text generated by AI tools.
- Parse planning metadata and shot-level information.
- Create and list video projects.
- Import storyboard shots into a project.
- Upload local image, keyframe, and video assets.
- Bind image, keyframe, and video asset paths to shots.
- Configure Nano Banana and Jimeng REST Adapter (Mock) provider settings without returning secrets.
- Generate Nano Banana images and keyframes through configured provider settings.
- Generate videos through the mock provider or Jimeng REST Adapter (Mock) workflow.
- Submit and poll Jimeng REST Adapter (Mock) video jobs for workflow testing.
- Validate shot timing, dependencies, missing assets, and duration alignment.
- Generate SRT and ASS subtitles.
- Build render plans from image/video assets.
- Reorder timeline shots and export render plans.
- Provide a Vue-based workbench UI for project selection, storyboard import,
  provider settings, asset library, shot timeline review, asset path entry,
  render pipeline control, and validation feedback.

## Out Of Scope For v1.0.0-beta.2

- Browser automation for external image or video generation tools.
- Background render queues.
- Cloud rendering.
- Account management.
- Multi-user collaboration.
- Real FFmpeg rendering.
- Subtitles or dubbing workflows beyond subtitle file generation.
- New providers, new AI models, new API capabilities, new pages, and new data tables
  during RC1 readiness preparation.
- Real Volcano Engine Jimeng API integration.

## API Contract

All FastAPI endpoints return a standard JSON envelope.

Successful responses:

```json
{
  "success": true,
  "data": {}
}
```

Error responses:

```json
{
  "success": false,
  "error": {
    "code": "bad_request",
    "message": "Human-readable error message."
  }
}
```

Provider settings use a unified public schema with `provider`, `enabled`,
`configured`, `credentials`, non-secret configuration fields, and `updated_at`.
Credential values are accepted on save but never returned.

## Jimeng Contract

The current Jimeng REST Adapter is a mock workflow-testing scaffold. It validates
settings, video job submission, polling, timeline updates, and render-plan
integration, but it does not connect to the real Volcano Engine Jimeng API.
Real Jimeng integration is planned for a future release.

## Technical Stack

- Backend: FastAPI, Pydantic, SQLite, pytest.
- Frontend: Vue 3, Vite, Vue Router, Vitest.
- Media tooling: FFmpeg/ffprobe integration points for duration probing and render planning.
- CI: GitHub Actions for backend tests, frontend tests, frontend build, and lint.

## Quality Requirements

- The project must remain independent and contain only AI Video Workbench code.
- No unrelated product routes, pages, state stores, databases, or assets should be included.
- Backend tests must pass with `python -m pytest tests/video_workbench -v`.
- Frontend tests must pass with `npm run test -- --run`.
- Browser E2E smoke must pass with `npm run test:e2e`.
- Frontend production build must pass with `npm run build`.
- Backend lint must pass with `ruff check .`.
- Frontend lint must pass with `npm run lint` or `eslint .`.
