# AI 短视频生产工作台设计

## 摘要

构建一个本地网页工作台，用来把 AI 短视频流程从分镜文本规模化推进到最终对齐的成片。系统会解析 Google AI Studio 输出的分镜批次，管理图片和视频生成任务，追踪素材状态，校验依赖关系和时间轴，生成字幕，并导出音频、画面、字幕三轨对齐的最终视频。

长期形态会包含 Google AI Studio、Nano banana 和即梦的浏览器 Agent。第一版实现聚焦稳定的本地 MVP：分镜解析、镜头管理、提示词包导出、素材导入、校验、字幕和 FFmpeg 渲染。浏览器自动化作为后续执行层接入，不作为项目状态的核心控制者。

## 目标

- 批量处理固定格式的 Google AI Studio 分镜输出。
- 支持项目级规划信息、批次级文本和镜头级提示词。
- 自动识别镜头类型：
  - 模式 A：基于底图的叠加/编辑。
  - 模式 B：全新构图。
  - 关键节点视频：先生成关键帧，再用关键帧 + I2V 提示词在即梦生成视频。
- 提供一个本地可视化工作台，用于状态追踪、审核、修正和重试。
- 管理生成图片、关键帧、视频、音频、字幕、报告和渲染结果。
- 保证最终导出前，视频、音频和字幕已经对齐。
- 为操作 Google AI Studio、Nano banana 和即梦的浏览器 Agent 预留清晰接入点。

## MVP 暂不实现的目标

- 跨所有网页工具的完全无人值守生成。
- 自动判断生成图片和视频的主观质量。
- 绕过登录、验证码、账号额度、平台限流或网页工具限制。
- 依赖 Nano banana 或即梦的非官方 API。

## 产品形态

产品是一个本地网页工作台，而不是单纯的命令行脚本。

工作台包含：

- 项目侧边栏：当前视频、角色设定卡、分镜批次、素材库和导出区域。
- 中央镜头时间线：每个镜头一张卡片。
- 选中镜头详情面板：展示时间、类型、台词、提示词、依赖、已生成素材和操作按钮。
- 进度栏：展示已解析镜头数、已完成图片数、已完成视频数、缺失素材数和是否可渲染。
- 任务日志：展示解析、导入、校验、渲染，以及后续浏览器 Agent 的执行活动。

这种形态是必要的，因为这个流程包含大量耗时、容易失败、且需要视觉主观判断的任务。用户需要看到状态、发现缺失或不合格素材、替换单个镜头，并在不丢失整个项目状态的情况下重新渲染。

## 输入模型

系统接受固定格式的 Google AI Studio 输出，分为三层。

### 项目级

示例：

- 音频总时长：538 秒。
- 预计常规图片：约 359 张。
- 预计关键节点：10 个。
- 预计批次数：约 24 批。
- 每批数量：15 个镜头。
- 角色设定卡和全局风格锁。

### 批次级

示例：

- 批次编号。
- 镜头范围，例如 `第 1 张 — 第 15 张`。
- 叙事阶段说明。
- 继续生成提示，例如要求用户回复 `继续`。

### 镜头级

每个镜头解析以下字段：

- 镜头编号。
- 开始时间。
- 结束时间。
- 持续时长。
- 模式 A 或 B。
- 关键节点标记。
- 关键节点类型。
- 画面风格说明。
- 输出形式。
- 如果存在，解析中文和英文台词。
- 图片提示词。
- I2V 提示词。
- 底图依赖。
- 保持不变指令。
- 新增元素指令。

## 镜头类型

### 模式 B：全新构图

数据流：

1. 使用全局角色/风格信息加当前镜头提示词。
2. 导出为 Nano banana 图片任务。
3. 导入生成后的图片。
4. 审核并标记完成。
5. 渲染时把图片转换成目标时长的视频片段。

### 模式 A：叠加/编辑

数据流：

1. 读取 `Base Image` 引用。
2. 等待被引用的底图存在。
3. 使用底图、保持不变指令和新增元素指令，导出为 Nano banana 编辑任务。
4. 导入生成后的图片。
5. 审核并标记完成。
6. 渲染时把图片转换成目标时长的视频片段。

校验器必须标记以下模式 A 镜头：底图缺失、底图尚未生成，或者底图编号晚于依赖它的镜头。

### 关键节点视频

已确认策略：

1. 先用 Nano banana 生成关键帧图片。
2. 在即梦中使用关键帧 + I2V 提示词。
3. 导入生成后的视频。
4. 渲染时校验视频，并把视频标准化到镜头目标时长。

这个策略可以提高画风一致性，并让关键节点和静态图片进入同一套素材管理模型。

## 核心模块

### 前端工作台

推荐技术栈：Vue 和 Vite，与当前项目方向一致。

职责：

- 创建和打开视频项目。
- 导入角色设定卡、音频、规划文本和分镜批次。
- 展示镜头时间线和镜头详情面板。
- 按批次、类型、状态、缺失素材、失败状态和依赖关系筛选。
- 预览图片、关键帧、视频、音频和字幕。
- 当解析器需要人工修正时，编辑解析字段。
- 导出提示词包。
- 通过拖拽或文件选择器导入生成素材。
- 触发校验和渲染。
- 展示校验报告和渲染报告。

### 后端服务

推荐技术栈：Python 和 FastAPI。

职责：

- 解析分镜文本。
- 存储项目和镜头数据。
- 管理项目文件。
- 生成提示词包。
- 将导入素材匹配到镜头。
- 生成字幕。
- 校验时间线和依赖关系。
- 运行 FFmpeg 渲染任务。
- 向前端暴露任务和状态 API。
- 为浏览器 Agent 提供扩展接入点。

### 浏览器 Agent 层

推荐技术栈：Playwright。

职责：

- AI Studio Agent：
  - 打开当前 AI Studio 对话。
  - 检测已经完成的分镜批次。
  - 抓取批次文本。
  - 发送 `继续` 或 `生成分镜` 等续写指令。
  - 遇到格式异常、重复范围、缺失范围、登录问题、验证码或账号问题时暂停。

- Nano banana Agent：
  - 执行模式 B 图片任务。
  - 执行模式 A 叠加/编辑任务。
  - 为关键节点视频镜头生成关键帧。
  - 下载文件并绑定到镜头 ID。
  - 遇到登录、验证码、额度限制、明显失败或用户要求审核时暂停。

- 即梦 Agent：
  - 根据关键帧和 I2V 提示词执行 I2V 任务。
  - 下载视频结果。
  - 记录实际时长和文件路径。
  - 遇到登录、验证码、额度限制、生成失败或用户要求审核时暂停。

浏览器 Agent 层不拥有项目真相。它只消费工作台任务，并把结果写回后端。

## 镜头状态机

镜头状态：

- `parsed`
- `waiting_for_base_image`
- `image_pending`
- `image_generating`
- `image_ready`
- `image_needs_review`
- `image_failed`
- `keyframe_pending`
- `keyframe_ready`
- `video_pending`
- `video_generating`
- `video_ready`
- `video_needs_review`
- `video_failed`
- `approved`
- `in_render_plan`
- `rendered`

失败或被驳回的镜头只回退到必要步骤。整个项目不应该因为一张图片或一个视频失败而需要重跑。

## MVP 功能

### 项目创建

用户可以创建视频项目并导入：

- 视频标题。
- 角色设定卡。
- 音频文件。
- Google AI Studio 规划文本。
- 一个或多个分镜批次文本。

### 分镜导入和解析

用户可以粘贴或导入批次文本。解析器会提取项目元数据、批次元数据和镜头记录。

解析结果可以在 UI 中审核和修正。

### 镜头时间线

工作台以每个镜头一张卡片的方式展示：

- 镜头编号。
- 时间范围。
- 类型。
- 状态。
- 缺失素材标记。
- 依赖标记。
- 审核标记。

### 提示词包导出

系统分别导出以下任务包：

- Nano banana 常规图片任务。
- Nano banana 关键帧任务。
- 即梦 I2V 任务。

每个任务包含镜头 ID、提示词、依赖文件、期望输出文件名和目标时长。

### 素材导入和匹配

用户可以把生成后的图片和视频拖入素材库。

匹配规则：

- 优先使用期望文件名，例如 `shot_001_image.png`、`shot_010_keyframe.png` 和 `shot_010_video.mp4`。
- 如果文件名存在歧义，要求用户手动绑定素材。
- 一旦绑定成功，更新镜头状态和校验状态。

### 校验

校验器检查：

- 镜头编号跳号。
- 镜头编号重复。
- 时间断档。
- 时间重叠。
- 解析出的镜头数量与项目规划预估是否一致。
- 解析出的关键节点数量与规划预估是否一致。
- 缺失图片素材。
- 缺失关键帧。
- 缺失视频素材。
- 模式 A 底图依赖。
- 视频时长不匹配。
- 字幕时间线覆盖情况。
- 最终渲染准备状态。

### 渲染

渲染器以音频作为主时间线。

规则：

- 最终视频时长必须在小容差范围内匹配音频时长。
- 图片镜头会转换成精确目标时长的视频片段。
- 视频镜头会通过裁切、保持最后一帧或受控变速，标准化到目标时长。
- 如果标准化会造成明显的视觉或时间问题，该镜头会被标记为需要审核。
- 当必要素材缺失，或时间线存在未解决的断档/重叠时，阻止渲染。

输出：

- `final_clean.mp4`
- `final_with_subtitles.mp4`
- 渲染报告。

### 字幕

字幕从每个镜头的台词字段生成。

MVP 支持输出：

- 中文字幕 SRT。
- 当存在英文台词时，输出英文 SRT。
- 中英双语 ASS 字幕文件。
- 烧录字幕版视频。
- 无烧录字幕的纯净视频。

字幕时间使用镜头开始和结束时间。如果字幕结束时间、视频结束时间和音频结束时间没有对齐，则阻止导出。

## 错误处理

错误应该可见、局部化，并且可恢复。

示例：

- 解析器无法识别时间范围：展示原始文本块，并允许人工修正。
- 模式 A 引用了缺失图片：标记为被依赖阻塞。
- 导入素材无法匹配：放入未分配素材区。
- 视频过短：根据配置阈值，保持最后一帧或要求审核。
- 音频和分镜时长不同：展示差值，并在解决前阻止最终导出。
- 浏览器 Agent 遇到验证码或登录问题：暂停任务并请求人工介入。

## 测试策略

MVP 测试应覆盖：

- 用真实分镜样例解析规划元数据、批次元数据、模式 A、模式 B 和关键节点文本块。
- 时间解析和持续时长计算。
- 检测断档、重叠、重复镜头和缺失依赖。
- 提示词包生成。
- 素材文件名匹配。
- SRT 和 ASS 字幕生成。
- 渲染计划生成。
- 在 mock 执行下构造 FFmpeg 命令。
- 项目创建、分镜导入、素材绑定、校验和触发渲染的 API 端点。

## 落地计划

### 阶段一：本地工作台 MVP

构建稳定的本地工作流：

1. 项目创建。
2. 分镜解析器。
3. 镜头数据库。
4. 时间线 UI。
5. 提示词包导出。
6. 素材导入和匹配。
7. 校验报告。
8. 字幕生成。
9. 音频、视频和字幕对齐渲染。

### 阶段二：AI Studio Agent

为 AI Studio 添加浏览器自动化：

1. 检测当前批次。
2. 抓取批次文本。
3. 继续生成。
4. 追加已解析批次。
5. 遇到异常输出时暂停。

### 阶段三：Nano banana 和即梦 Agent

为素材生成添加浏览器自动化：

1. 在 Nano banana 中生成图片和关键帧。
2. 在即梦中生成 I2V 视频。
3. 下载并绑定素材。
4. 通过工作台从失败中恢复。

## MVP 默认策略

- 烧录字幕使用清晰的底部居中 ASS 样式；启用双语输出时，中文在上，英文在下。
- 视频素材过短时，默认保持最后一帧补齐。如果缺失时长大到看起来别扭，校验器会在最终导出前标记该镜头需要审核。
- 视频素材过长时，默认裁切到目标镜头时长。后续版本可以增加用户自选入点/出点。
- 项目状态存储在 SQLite 中，便于可靠查询和状态更新；同时把生成的项目产物写成 JSON、字幕、提示词、报告和媒体文件，保证可迁移性。
- 工作台应作为本仓库中的独立应用区域构建，而不是混进任何既有产品页面。

## 已确认方向

已确认的方向是：

- 以本地网页工作台作为主要产品界面。
- 在 MVP 中包含音频、画面和字幕对齐。
- 将浏览器 Agent 作为后续执行插件接入。
- 对于关键节点视频镜头，先生成 Nano banana 关键帧，再用该关键帧和 I2V 提示词在即梦中生成视频。

当前 beta 能力包括：

- 创建项目和查看项目列表。
- 将分镜文本导入为持久化镜头记录。
- 上传素材、查看素材库，并绑定 image、keyframe、video 路径。
- Nano Banana Provider Settings、图片生成和关键帧生成。
- Jimeng Provider Settings、Mock/Jimeng 视频生成，以及 Jimeng REST 任务提交/轮询流程。
- Timeline 重排、Timeline 持久化、Render Plan 生成和导出。
- SRT/ASS 字幕生成工具和媒体探测辅助模块。
- Vue 工作台面板：Provider Settings、Asset Library、Shot Timeline、Validation、Render Pipeline、Timeline Editing、Video Jobs。

API 响应使用统一 envelope。

成功响应：

```json
{
  "success": true,
  "data": {}
}
```

错误响应：

```json
{
  "success": false,
  "error": {
    "code": "bad_request",
    "message": "可读错误信息。"
  }
}
```

Nano Banana 和 Jimeng Provider Settings 使用统一公开 schema：

- `provider`
- `enabled`
- `configured`
- `credentials`，仅包含是否已配置的布尔值
- 非密钥字段，例如 `base_url`、`region`、`endpoint`、`model`
- `updated_at`

密钥值可以在保存时提交，但 API 响应绝不能回传密钥明文。

Beta Hardening Sprint 2 明确不包含：新增 Provider、新增 AI 模型、新增 API 能力、新页面、新数据表、现有 REST 工作流形态之外的真实 Jimeng 集成、字幕/配音产品工作流、ffmpeg 渲染，以及新的 AI 能力。

CI 必须通过后端测试、前端测试、前端构建和 lint gate（`ruff check .` 与 `eslint .`）。
